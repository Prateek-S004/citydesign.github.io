Smart City Search Engine with Service Recommendations

Description

This Smart City Search Engine enables users to search for city services based on patterns in names and allows sorting by lexicographic order, rating, and proximity. It uses a menu-driven interface for input and displays sorted results based on user preferences.

Data Structures and Algorithms

Data Structures:

  • Vector of Tuples: Stores services with name, coordinates, and rating.
  • Strings and Tuples: Handle service names and related data.

Algorithms:

  • Brute-Force String Search (BFSS): For matching service names with patterns.
  • Insertion Sort: For sorting services lexicographically.
  • Merge Sort: For sorting services by rating.
  • QuickSort: For sorting services by proximity using Euclidean distance.

Why This Should Be Addressed

  • Personalized Service Recommendations: Helps users find relevant services based on their preferences.
  • User Convenience: Multiple sorting options enhance the user experience.
  • Smart City Optimization: Facilitates better access to services in a smart city environment.

Efficiency Analysis

Brute-Force String Search (BFSS):

  • Time Complexity: O(n * m), where n is the length of the text and m is the length of the pattern.

Insertion Sort:

  • Time Complexity: O(n2), inefficient for large datasets.

Merge Sort:

  • Time Complexity: O(n log n).
  • Space Complexity: O(n).

QuickSort:

  • Time Complexity: O(n log n) on average, O(n2) in the worst case.
  • Space Complexity: O(log n).

Euclidean Distance Calculation:

  • Time Complexity: O(1) per calculation.

Conclusion

The search engine provides an efficient way to find and sort services but would be more effective with optimizations for larger datasets. It is designed to scale with the growing demands of smart cities.
